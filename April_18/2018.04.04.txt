> 모든 이미지를 html markup 에 넣어도 됨. 

** html 에 이미지를 넣게 되는 경우 
- 의미가 있다면 alt 테그에 내용을 넣고 
- 의미가 없다면/장식의 의미라면 alt 필요 없음

** css 백그라운드에 넣게되는 경우
html 에 넣게되면 이미 골격이 세팅이 된 상태라 같은 html 골격으로 다양한 웹사이트의 효과를 주기 위해서는 
텍스트는 html 에 그리고 css 에 이미지를 삽입한다. 
- 배경 처리한 이미지는 alt 대신 aria- 테그를 사용하면 해결할 수 는 있다.

*** 이미지 배경 처리하게 되면 배경 크기에 맞게 이미지가 들어가므로 
반응형 웹사이트 디자인에서는 배경처리된 이미지가 헨들링하기 어렵다.

-> 의미가 있는 이미지들은 마크업에 넣어주고 alt 테그를 달고
큰 의미가 없는 이미지는 css 배경 처리해주고, 묶음으로 동일하게 한 번에 바껴야 하는 이미지들은 css 에서 바로 
이미지 경로 수정해서 바꿔줄 수 있다.
